Name des Quellprojekts, aus dem Artefakte kopiert werden sollen. Darf Referenzen auf Buildparameter, wie z.B. $PARAM, enthalten (Bemerkung: Wenn ein Parameter benutzt wird, müssen alle authentifizierten Benutzer Zugriff auf das Quellprojekt haben; das verhindert die Benutzung von Parametern um auf Artefakte von private Jobs zuzugreifen).

Wenn ein Maven Projekt angegeben wird, werden die Artefakte aus allen Modulen dieses Projekts kopiert. Mit JOBNAME/MODULENAME kann ein einzelnes Modul angegeben werden; per copy und paste kann man diesen Wert aus der URL für dieses Modul übernehmen.
Beispiel: MyMavenJob/my.group$MyModule

Wenn ein Multikonfigurations-(Matrix-)Projekt angegeben wird, werden die Artefakte aus allen Konfigurationen jeweils in ein Unterverzeichnis mit dem Namen der Konfiguration, wie er in der URL zu sehen ist, kopiert.
Beispiel: Wenn das Zielverzeichnis fromMatrix heißt dann werden $WORKSPACE/fromMatrix/label=slaveA/dist/mybuild.jar und $WORKSPACE/fromMatrix/label=slaveB/dist/mybuild.jar angelegt.
Um die Artefakte aus einer einzelnen Konfiguration zu kopieren, muss JOBNAME/AXIS=VALUE,.., wie es in der URL für diese Konfiguration zu sehen ist, angegeben werden.
Beispiel: MyMatrixJob/jdk=Java6u17
Wenn Artefakte von einem Matrixprojekt zu einem anderen Matrixprojekt kopiert werden sollen, kann ein Parameter benutzt werden um die richtige Konfiguration im Quellprojekt auszuwählen. Beispiel: OtherMatrixJob/jdk=$jdk
